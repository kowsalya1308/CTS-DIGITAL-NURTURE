💾 Manage Your Preferences

This page demonstrates how to save and retrieve user preferences using browser storage:

  • localStorage: Persists data even after browser closes (long-term storage)
  • sessionStorage: Data is cleared when browser tab closes (session-only)
  • JSON serialization: Convert objects to strings for storage

📚 Understanding Storage Types

localStorage

Persistence: Data remains even after closing the browser and reopening later

Scope: Shared across all tabs and windows of the same origin

Capacity: Typically 5-10MB per origin

Use Case: Saving user preferences, theme settings, saved form data

// Example usage:
localStorage.setItem('favoriteEvent', 'festival');
const event = localStorage.getItem('favoriteEvent');
localStorage.removeItem('favoriteEvent');
localStorage.clear(); // Clear all data

sessionStorage

Persistence: Data is cleared when the tab/window is closed

Scope: Isolated to each tab or window

Capacity: Typically 5-10MB per origin

Use Case: Temporary session data, form progress, temporary settings

// Example usage:
sessionStorage.setItem('formProgress', JSON.stringify(data));
const data = JSON.parse(sessionStorage.getItem('formProgress'));
sessionStorage.removeItem('formProgress');

Differences

Feature localStorage sessionStorage
Expiration Never (unless cleared) When tab closes
Shared Across Tabs Yes No
Access Persistent Current session only

💡 Best Practices

  • Use JSON.stringify(): Convert objects to strings before storing
  • Use JSON.parse(): Convert strings back to objects when retrieving
  • Check for errors: Storage might be full or disabled in incognito mode
  • Validate data: Always validate retrieved data before using
  • Respect user choice: Don't store sensitive data like passwords
