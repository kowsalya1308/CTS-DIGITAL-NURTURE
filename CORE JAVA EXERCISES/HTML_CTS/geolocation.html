🛰️ Geolocation API Features

This exercise demonstrates the Geolocation API with the following features:

  • getCurrentPosition(): Gets the user's current location
  • enableHighAccuracy: Option for more precise location (uses GPS)
  • timeout: Maximum time to wait for location (in milliseconds)
  • maximumAge: How long to cache the position (in milliseconds)
  • Error Handling: Handles permission denial, timeout, and position unavailable
  • Distance Calculation: Uses Haversine formula to calculate distance between points

Code Example

const options = {
  enableHighAccuracy: true,
  timeout: 10000,
  maximumAge: 0
};

navigator.geolocation.getCurrentPosition(
  successCallback,
  errorCallback,
  options
);

Error Codes

  • PERMISSION_DENIED (1): User denied location permission
  • POSITION_UNAVAILABLE (2): Location information unavailable
  • TIMEOUT (3): Timeout waiting for position

🔒 Privacy & Security

  • The Geolocation API requires HTTPS on production sites (HTTP allowed for localhost)
  • Users must explicitly grant permission for each website
  • Browsers typically show a permission prompt with options to allow, block, or remember choice
  • Location data is handled on the client-side in this exercise (not sent to server)
  • Always include clear explanations of why you need location data
